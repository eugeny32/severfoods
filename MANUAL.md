## 1. Общее описание системы

**СеверФудс** — система учёта питания сотрудников предприятия ООО «Север». Система состоит из двух компонентов:

| Компонент | Назначение |
|-----------|------------|
| **Онлайн** (веб-приложение) | Центральный сервер: база данных сотрудников, отчёты, управление пользователями, чат |
| **Оффлайн** (Electron-приложение для Windows) | Автономная точка питания: сканирование QR-карт, локальная БД, периодическая синхронизация с сервером |

**Принцип работы:**

1. Администратор регистрирует сотрудников в онлайн-системе и печатает им QR-карты.
2. На точке питания установлено оффлайн-приложение. При входе сотрудник прикладывает QR-карту к сканеру.
3. Оффлайн-приложение фиксирует приём пищи в локальную базу данных.
4. По расписанию (или вручную) локальные данные синхронизируются с центральным сервером.
5. В онлайн-системе руководитель просматривает отчёты и статистику питания.

## 2. Онлайн-система (веб-приложение)

Адрес: **https://www.severfoods.ru**

### 2.1 Роли пользователей

| Роль | Описание | Доступные разделы |
|------|----------|-------------------|
| **Оператор** | Сотрудник на точке приёма (кассир, вахтёр) | Главная (сканер), Чат |
| **Администратор** | Управляет сотрудниками и данными одного объекта | Главная, Сотрудники, Отчёты, Чат, QR-карты, Точки питания |
| **Супер-администратор** | Полный доступ ко всем объектам и пользователям | Все разделы + Пользователи + Настройки системы |

### 2.2 Вход в систему

1. Откройте браузер и перейдите по адресу системы.
2. Введите **логин** (e-mail или имя пользователя) и **пароль**.
3. Нажмите «Войти».

> При ошибке входа убедитесь, что включена правильная раскладка клавиатуры и Caps Lock выключен.

Для смены пароля обратитесь к супер-администратору.

### 2.3 Главная страница — Сканер QR

Главная страница является рабочим местом оператора для регистрации приёмов пищи через веб-браузер.

**Элементы страницы:**

- **Поле ввода QR-кода** — сканер подключается как HID-клавиатура; считанный код вводится автоматически.
- **Кнопка «Сканировать»** или нажатие **Enter** — отправляет код на проверку.
- **Результат валидации** — всплывающее сообщение:
  - *Зелёный фон* — приём зафиксирован, выводятся ФИО и организация сотрудника.
  - *Оранжевый фон* — повторный приём (сотрудник уже питался сегодня).
  - *Красный фон* — сотрудник не найден или заблокирован.
- **Статистика дня** — счётчики приёмов пищи по типам (завтрак, обед, ужин).
- **Журнал сканирований** — список последних событий с временными метками.

**Кнопка «Чат»** в навигации отображает число непрочитанных сообщений красным бейджем.

### 2.4 Страница «Сотрудники»

Управление базой сотрудников, допущенных к питанию.

**Возможности:**

- Поиск сотрудника по ФИО, табельному номеру или организации.
- Добавление нового сотрудника (кнопка «Добавить»).
- Редактирование данных: ФИО, организация, подразделение, тип питания, статус (активен/заблокирован).
- Блокировка/разблокировка — заблокированный сотрудник получает отказ при сканировании.
- Печать QR-карты для выбранного сотрудника.
- **Экспорт в Excel** — выгрузка сводного отчёта по всем сотрудникам (количество приёмов, дней в столовой, группировка по организациям).

**Поля карточки сотрудника:**

| Поле | Описание |
|------|----------|
| ФИО | Полное имя |
| Организация | Наименование работодателя |
| Подразделение | Отдел / бригада |
| Тип питания | Завтрак / Обед / Ужин / Комбо |
| Статус | Активен / Заблокирован |
| QR-код | Уникальный идентификатор, генерируется автоматически |

### 2.5 Страница «Отчёты»

Аналитика приёмов пищи за выбранный период.

**Фильтры:**

- Период (дата «с» — дата «по»)
- Точка питания
- Организация
- Тип приёма (завтрак, обед, ужин)

**Доступные отчёты:**

| Отчёт | Описание |
|-------|----------|
| Сводный по дням | Количество приёмов по каждому дню периода |
| По сотрудникам | Сколько раз питался каждый сотрудник |
| По организациям | Сводка по организациям с подытогами |
| Детальный журнал | Каждый приём пищи с временной меткой и точкой |

**Экспорт** — все отчёты можно выгрузить в формате Excel (.xls, SpreadsheetML).

### 2.6 Страница «Чат»

Корпоративный мессенджер внутри системы.

**Типы комнат:**

| Тип | Описание | Кто может писать |
|-----|----------|-----------------|
| **Личное сообщение (DM)** | Диалог между двумя пользователями | Оба участника |
| **Группа** | Чат для нескольких участников | Все участники |
| **Канал** | Публичный канал объявлений | Все участники |

**Работа с чатом:**

1. Выберите комнату в левой панели.
2. Введите сообщение в поле внизу и нажмите **Enter** или кнопку «Отправить».
3. Для создания новой комнаты нажмите «+» рядом с заголовком «Чаты».
4. Для добавления участников откройте настройки комнаты (шестерёнка).

**Уведомления:**

- На кнопке «Чат» в навигации главной страницы отображается красный бейдж с числом непрочитанных сообщений.
- При получении нового сообщения в группе/канале — всплывающее окно (тост) в нижнем левом углу с именем отправителя и текстом сообщения.
- Звуковой сигнал (трёхнотный перезвон) при каждом новом сообщении.
- В личных сообщениях — только звуковой сигнал (без тоста).

### 2.7 Страница «QR-карты»

Печать QR-карт для сотрудников.

- Выберите одного или нескольких сотрудников.
- Нажмите «Печать выбранных» или «Печать всех».
- Откроется страница предпросмотра печати с QR-кодами.
- QR-карты содержат ФИО, организацию и уникальный код сотрудника.

> Рекомендуется ламинировать QR-карты для длительного использования.

### 2.8 Страница «Точки питания»

Управление точками питания (местами установки оффлайн-приложения).

- Добавление / редактирование / удаление точек.
- Каждой точке присваивается имя и токен синхронизации.
- Токен необходим для настройки оффлайн-приложения на этой точке.

### 2.9 Страница «Настройки»

Доступна супер-администратору.

- Настройки отображения и работы системы.
- Управление токенами синхронизации для оффлайн-точек.
- Системные параметры (типы питания, рабочие часы).

### 2.10 Страница «Пользователи»

Доступна супер-администратору.

- Список всех пользователей системы.
- Создание нового пользователя: логин, пароль, роль, привязка к точкам питания.
- Редактирование и блокировка учётных записей.

### 2.11 Уведомления и звук

Система автоматически проверяет новые сообщения каждые **30 секунд**.

Браузер должен разрешить воспроизведение звука для сайта. При первом посещении браузер может запросить подтверждение — нажмите «Разрешить».

## 3. Оффлайн-система (настольное приложение)

### 3.1 Архитектура оффлайн-приложения

- **Технология:** Electron (Node.js + Chromium), работает как настольное приложение Windows.
- **Локальная БД:** SQLite-файл `severfoods.db`, хранится в `%APPDATA%\SeverFoods\` — **не удаляется при обновлении или переустановке**.
- **Веб-сервер:** Встроенный Express-сервер на порту `3847`, работает только локально.
- **Синхронизация:** Периодические HTTPS-запросы к центральному серверу СеверФудс.

**Расположение файлов:**

| Файл / Папка | Путь | Описание |
|---|---|---|
| Исполняемый файл | `%LOCALAPPDATA%\SeverFoods\SeverFoods.exe` | Основной файл приложения |
| Конфигурация | `%LOCALAPPDATA%\SeverFoods\.env` | Токен и адрес сервера |
| База данных | `%APPDATA%\SeverFoods\severfoods.db` | Локальные данные (НЕ удаляется) |

### 3.2 Установка и первоначальная настройка

**Требования:** Windows 10 / 11, x64, права пользователя (права администратора НЕ требуются).

**Шаги установки:**

1. Запустите файл `SeverFoods-Setup-1.3.0.exe`.
2. Нажмите «Далее» на экране приветствия.
3. При необходимости измените папку установки (по умолчанию `%LOCALAPPDATA%\SeverFoods`).
4. На экране **«Токен синхронизации»** введите токен, выданный администратором сервера.
   - Поле «URL сервера» оставьте по умолчанию (`https://www.severfoods.ru`), если не используется локальный сервер.
5. Нажмите «Далее» → «Установить» → «Завершить».
6. Приложение запустится автоматически (если оставлен флажок «Запустить СеверФудс»).

**При первом запуске без токена** (например, запуск `SeverFoods.exe` напрямую без прохождения установки) откроется окно настройки. Введите токен и нажмите «Сохранить».

### 3.3 Обновление приложения

**Обновление сохраняет:**
- Токен синхронизации (`.env` обновляется только если введён новый токен)
- Базу данных с историей питания (`%APPDATA%\SeverFoods\`)
- Журнал приёмов пищи

**Процедура обновления:**

1. Закройте работающее приложение (правой кнопкой на иконке в трее → «Выход»).
2. Запустите новый инсталлятор `SeverFoods-Setup-X.X.X.exe`.
3. На экране «Токен синхронизации» **оставьте поле пустым** — текущий токен сохранится автоматически.
   - Если нужно изменить токен — введите новый.
4. Завершите установку. Данные в базе сохранены.

> **Важно:** Никогда не удаляйте папку `%APPDATA%\SeverFoods\` вручную — в ней находится база данных со всей историей питания.

### 3.4 Страница «Сканер»

Основная рабочая страница оператора точки питания.

**Элементы интерфейса:**

- **Поле QR-кода** — сканер (подключённый как USB HID) вводит код автоматически.
- **Кнопка «Сканировать»** / клавиша **Enter** — ручная отправка кода.
- **Тип питания** — выпадающий список (Завтрак / Обед / Ужин / Авто).
- **Всплывающий баннер** — результат валидации (см. раздел 3.6).
- **Статистика сессии** — счётчики сканирований за текущий день.
- **Журнал** — последние события в хронологическом порядке.
- **Счётчик очереди** — отображается при накоплении кодов в очереди.

### 3.5 Очередь QR-кодов

При потоковом сканировании (несколько карт подряд) коды выстраиваются в очередь FIFO и обрабатываются последовательно с паузой 250 мс между ними.

**Принципы работы очереди:**

- Одинаковые коды в очереди не дублируются (защита от «дрожания» считывателя).
- Следующий код обрабатывается сразу после завершения текущего.
- Отображение результата предыдущего кода **не блокирует** приём новых кодов.
- Число кодов, ожидающих обработки, показывается на экране.

### 3.6 Цветовые сигналы валидации

| Цвет баннера | Значение | Время показа |
|---|---|---|
| **Ярко-зелёный** | Приём пищи успешно зафиксирован | 3 секунды |
| **Оранжевый** | Повторный приём (уже питался сегодня) | 3 секунды |
| **Ярко-красный** | Отказ: не найден в базе, заблокирован или ошибка | 8 секунд |

**Приоритет отображения:**

- Красный баннер (отказ) **не может быть прерван** зелёным или оранжевым.
- Зелёный и оранжевый баннеры могут сменять друг друга.
- Это гарантирует, что оператор всегда увидит отказ, даже при быстром потоке карт.

### 3.7 Звуковые сигналы

| Событие | Сигнал | Длительность |
|---|---|---|
| Успешный приём | Мелодичный перезвон (три ноты) | ~1 секунда |
| Повторный приём | Двойной звуковой сигнал | ~0.5 секунды |
| Отказ | Тревожный прерывистый сигнал (880/660 Гц) | **5 секунд** |

Тревожный сигнал при отказе звучит в течение 5 секунд, не прерывая обработку следующих QR-кодов в очереди.

> Для работы звука необходимо, чтобы звуковая карта Windows была включена и громкость не была на минимуме.

### 3.8 Страница «Сотрудники»

Локальный список сотрудников, загруженный при последней синхронизации.

- Поиск по ФИО и организации.
- Просмотр данных сотрудника: организация, подразделение, тип питания, статус.
- Сотрудников **нельзя редактировать или добавлять** в оффлайн-приложении — изменения вносятся в онлайн-системе и загружаются при синхронизации.

### 3.9 Страница «Журнал»

Полный журнал всех приёмов пищи, зафиксированных на данной точке.

**Фильтры:**

- По дате (дата «с» — дата «по»)
- По сотруднику (поиск по имени)
- По типу питания

**Данные записи журнала:**

- Дата и время
- ФИО сотрудника
- Организация
- Тип питания
- Статус (Успешно / Повтор / Отказ)
- Синхронизировано с сервером (да/нет)

### 3.10 Синхронизация с сервером

Синхронизация выполняет два действия:
1. **Отправка** накопленных записей питания на центральный сервер.
2. **Загрузка** актуального списка сотрудников с сервера.

**Запуск синхронизации:**

- **Автоматически** — по расписанию (интервал настраивается).
- **Вручную** — кнопка «Синхронизировать» в интерфейсе приложения.
- **Из трея** — правой кнопкой на иконке → «Синхронизировать».

**Статус синхронизации** отображается в нижней строке приложения:
- `Синхронизировано: ДД.ММ.ГГГГ ЧЧ:ММ` — последний успешный сеанс.
- `Ошибка синхронизации` — нет связи с сервером или неверный токен.
- `Не синхронизировалось` — первый запуск или синхронизация ещё не выполнялась.

> При отсутствии связи с сервером приложение продолжает работу в полностью автономном режиме. Все данные сохраняются и будут отправлены при восстановлении соединения.

### 3.11 Настройки (токен и сервер)

Открываются через меню «Настройки» или иконку трея → «Настройки».

**Параметры:**

| Параметр | Описание |
|---|---|
| **Токен синхронизации** | Уникальный ключ для доступа к API сервера. Выдаётся администратором в онлайн-системе. |
| **URL сервера** | Адрес центрального сервера. По умолчанию: `https://www.severfoods.ru` |

После сохранения настроек выполняется немедленная синхронизация для проверки соединения.

### 3.12 Иконка в системном трее

Приложение сворачивается в системный трей (область уведомлений Windows) при нажатии кнопки закрытия окна.

**Меню трея (правая кнопка мыши):**

| Пункт | Действие |

## 4. Администрирование

### 4.1 Управление сотрудниками

**Добавление нового сотрудника:**

1. В онлайн-системе перейдите в раздел «Сотрудники».
2. Нажмите «Добавить сотрудника».
3. Заполните обязательные поля: ФИО, организация.
4. Сохраните. QR-код генерируется автоматически.
5. Напечатайте QR-карту (раздел «QR-карты» или кнопка «Печать» в карточке).
6. Выдайте карту сотруднику.
7. **Синхронизируйте** оффлайн-точки питания — новый сотрудник появится в локальной базе.

**Блокировка сотрудника:**

1. Найдите сотрудника в списке.
2. Нажмите «Редактировать».
3. Измените статус на «Заблокирован».
4. Сохраните и выполните синхронизацию оффлайн-точек.

После синхронизации при попытке сканирования карты заблокированного сотрудника оффлайн-приложение выдаст красный сигнал отказа.

### 4.2 Управление точками питания

Каждая точка питания (столовая, буфет) настраивается через раздел «Точки питания» в онлайн-системе.

**Создание точки:**

1. «Точки питания» → «Добавить».
2. Введите название точки.
3. Система генерирует уникальный **токен синхронизации**.
4. Скопируйте токен и введите его при установке оффлайн-приложения на этой точке.

> Один токен — одна точка питания. Не используйте один токен на нескольких компьютерах.

### 4.3 Токен синхронизации

Токен — это секретный ключ, связывающий оффлайн-точку с аккаунтом на сервере.

- Хранится в файле `%LOCALAPPDATA%\SeverFoods\.env` в строке `OFFLINE_SYNC_TOKEN=...`
- **Не передавайте токен третьим лицам.**
- При компрометации токена — сгенерируйте новый в онлайн-системе и обновите настройки оффлайн-приложения.
- Токен сохраняется при обновлении приложения автоматически.

### 4.4 Резервное копирование

**Что резервировать:**

| Данные | Расположение | Метод |
|---|---|---|
| Онлайн-БД | На сервере хостинга | Автоматические бэкапы хостинга |
| Оффлайн-БД | `%APPDATA%\SeverFoods\severfoods.db` | Копирование файла |
| Конфигурация | `%LOCALAPPDATA%\SeverFoods\.env` | Копирование файла |

**Расписание резервного копирования оффлайн-базы:**

Рекомендуется раз в неделю копировать файл `severfoods.db` на сетевой диск или USB-накопитель.

## 5. Часто задаваемые вопросы (FAQ)

**Q: Сотрудник потерял QR-карту. Что делать?**

A: В онлайн-системе откройте карточку сотрудника и нажмите «Перевыпустить QR-код» (или «Печать новой карты»). Старый QR-код станет недействительным автоматически после синхронизации.

---

**Q: Оффлайн-приложение не синхронизируется. Что проверить?**

1. Проверьте подключение к интернету.
2. Откройте «Настройки» в приложении и убедитесь, что токен введён верно.
3. Попробуйте вручную нажать «Синхронизировать».
4. Если ошибка повторяется — обратитесь к администратору для проверки токена в онлайн-системе.

---

**Q: Приложение не издаёт звуковых сигналов.**

A: Проверьте:
- Громкость Windows не на минимуме.
- Приложение не находится в списке заблокированных по звуку (Windows → Настройки звука → Параметры микшера).
- Звуковое устройство подключено и работает.

---

**Q: Как перенести приложение на другой компьютер?**

1. Установите приложение на новый ПК с тем же токеном.
2. Скопируйте файл `severfoods.db` из `%APPDATA%\SeverFoods\` старого ПК в аналогичную папку на новом.
3. Запустите приложение — история питания будет восстановлена.

---

**Q: Можно ли использовать приложение без интернета?**

A: Да. Приложение работает полностью автономно. Все приёмы пищи сохраняются в локальной базе и будут синхронизированы с сервером, как только появится соединение.

---

**Q: Красный баннер «Не найден в базе». Что это значит?**

A: Возможные причины:
- Сотрудник не был добавлен в онлайн-систему.
- Сотрудник добавлен, но синхронизация ещё не выполнялась.
- Сотрудник заблокирован.
- QR-код повреждён или считан с ошибкой.

Выполните синхронизацию и повторите попытку. Если ошибка остаётся — проверьте карточку сотрудника в онлайн-системе.

---

**Q: Оба бейджа показывают «0 непрочитанных», хотя сообщения есть.**

A: Система учитывает только сообщения, пришедшие **после** открытия страницы. Обновите страницу, чтобы пересчитать счётчик с текущего момента.
