# SeverFoods Offline Changelog

## v1.5.2 (2026-07-02)

### Автоматическая миграция со старой установки
- В 1.5.1 путь установки был исправлен на per-user (`%LOCALAPPDATA%`), но если ставить
  новую версию поверх старой (в `C:\Program Files\`) не удаляя её — NSIS мог унаследовать
  старый путь, и ошибка `EPERM ... Program Files\SeverFoods\.env` возвращалась
- Теперь установщик сам обнаруживает предыдущую установку в реестре (HKLM) и тихо удаляет
  её перед установкой новой версии — ручное удаление через «Параметры Windows» больше
  не требуется, апгрейд «поверх» старой версии происходит корректно сам

## v1.5.1 (2026-07-02)

### Исправления установки
- `EPERM: ... open 'C:\Program Files\SeverFoods\.env'` при запуске после установки —
  установщик по умолчанию ставил приложение в `Program Files` (нужны права администратора
  на запись), а само приложение запускается без повышения прав и пишет `.env` в свою папку.
  Установка теперь по умолчанию идёт в `%LOCALAPPDATA%\Programs\SeverFoods\` (доступно
  на запись без администратора), как и было задумано изначально
- Кракозябры вместо кириллицы на странице лицензии установщика — `LICENSE.txt` был без
  BOM, из-за чего NSIS иногда читал UTF-8 текст как ANSI/CP1251; добавлен UTF-8 BOM

## v1.5.0 (2026-07-02)

### Автообновление
- Приложение само проверяет наличие новой версии при наличии интернета (раз в час,
  плюс сразу при запуске) и тихо скачивает её в фоне — без прерывания работы
- Установка (перезапуск с новой версией) происходит автоматически только когда
  компьютер простаивает 5+ минут подряд (нет ввода с клавиатуры/мыши) — не мешает
  оператору посреди сканирования
- Настройки → «Обновления»: текущая версия, статус проверки/загрузки, кнопки
  «Проверить сейчас» и «Установить сейчас» (немедленный перезапуск с обновлением)
- Пункт «Проверить обновления» в меню трея

## v1.4.0 (2026-07-01)

### Часовой пояс
- Настройки → «Часовой пояс»: явный выбор UTC-офсета точки (admin/super_admin)
- Счётчики "сегодня" (завтрак/обед/ужин/ночное) и журнал сканирования теперь считают
  местные сутки по этому офсету, а не по часовому поясу ОС/UTC — устранено расхождение
  с сайтом на точках в других часовых поясах (например UTC+7)
- Отображаемое время прохода в журнале и истории теперь считается явно по офсету,
  а не через парсинг UTC-строки как локального времени браузера

### Синхронизация
- Таймаут HTTP-запросов увеличен: 15с → 45с (обычные запросы), 120с (отправка накопленных
  офлайн-записей) — устранены ложные ошибки синхронизации на медленном интернете
- Отправка офлайн-записей теперь идёт пакетами по 200 штук вместо одного большого запроса —
  обрыв соединения не отменяет уже переданный прогресс

## v1.3.0 (2026-06-13)

### QR-очередь
- Замена `scanCooldown` на FIFO-очередь: сканы при обработке идут в очередь, а не игнорируются
- Дубли одного кода в очереди пропускаются (защита от быстрой камеры)
- Счётчик очереди отображается в попапе результата

### Всплывающее окно — полный редизайн
- Большой баннер снизу на всю ширину контентной зоны
- Ярко-зелёный фон = OK, оранжевый = повтор, ярко-красный = отказ/блокировка
- Крупный шрифт: 56px иконка, 30px имя, 20px сообщение
- Красный держится 8 секунд, остальные 3 секунды
- Приоритет: ошибка не перебивается успешными/повторными сканами

### Звук
- Отказ: 5-секундная тревога (чередующиеся тоны 880/660 Гц), не блокирует очередь
- OK: двухнотный восходящий сигнал (sine)
- Повтор: двухнотный нейтральный сигнал (triangle)
- Новая ошибка перезапускает тревогу с начала

### Увеличенный интерфейс для слабовидящих
- Цифры статистики: 32px (было 22px)
- QR-поле ввода: 17px, увеличенные отступы
- Кнопки навигации: 14px, больше padding
- Строки журнала сканирования: 14px
- Заголовки страниц: 22px
